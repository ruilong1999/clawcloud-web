## 需求一：价格调整

### 问题分析
当前定价配置（第 692 行）：
- vCPU: ¥0.27/小时
- 内存: ¥0.03/GB/小时
- 存储: ¥0.15/GB/月

按此定价计算一个 6G 内存的实例：
- 每月费用：6GB × ¥0.03 × 24小时 × 30天 = ¥12.96

这确实过于便宜，不符合主流云服务商的实际定价水平。

### 调整方案
参考 AWS/阿里云等主流云服务商的容器实例定价，调整为：

| 资源 | 原价格 | 新价格 | 说明 |
|------|--------|--------|------|
| vCPU | ¥0.27/小时 | ¥0.50/小时 | 约 85% 涨幅 |
| 内存 | ¥0.03/GB/小时 | ¥0.15/GB/小时 | 5倍涨幅 |
| 存储 | ¥0.15/GB/月 | ¥0.50/GB/月 | 约 3.3倍涨幅 |

### 修改点
1. `AZURE_PRICING` 常量（第 692 行）
2. 计算器页面的定价说明（第 408-412 行）
3. 各套餐的定价显示（第 432-449 行）
4. DEVLOG.md 中的定价参考（第 98-102 行）

### 调整后效果示例
- 基础版 (2核4G): ¥0.50×2 + ¥0.15×4 = ¥1.60/小时
- 标准版 (4核8G): ¥0.50×4 + ¥0.15×8 = ¥3.20/小时
- 专业版 (8核16G): ¥0.50×8 + ¥0.15×16 = ¥6.40/小时

## 需求二：安全配置 (HTTPS)

### 问题分析
当前 nginx.conf 只监听 80 端口（HTTP），浏览器会提示"不安全"。

### 解决方案
使用 Let's Encrypt 免费证书配置 HTTPS。

### 前提条件
- 需要有域名
- 服务器 80 和 443 端口开放
- 安装 certbot

### 修改步骤

#### 1. 更新 nginx.conf 添加 HTTPS 支持
- 添加 SSL 证书配置
- 配置 HTTP 到 HTTPS 重定向
- 添加安全响应头

#### 2. 更新 DEVLOG.md 部署说明
- 添加 HTTPS 配置步骤
- 添加 certbot 安装和证书获取说明
- 添加证书自动续期配置
